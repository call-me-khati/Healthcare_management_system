How It Works
Appointment Flow
1
Doctor Sets Hours
Define working days & times. System auto-creates 10-min slots.
2
Student Books
Views ✅ available and ❌ booked slots. Selects time, sets priority.
3
Queue Assigned
Queue number issued. Emergency cases rise to top automatically.
4
Doctor Consults
Writes diagnosis, prescriptions, lab requests, follow-up date.
5
All Notified
Patient gets lab request notification. Doctor gets lab-done alert. Follow-up reminders sent automatically.
Quality Assurance
Bugs Found & Fixed

4 logic bugs were identified and corrected during the final audit pass.

✅ Fix 1 — StudentController Missing Import
UserModel was used in handleProfile() without being required. Added require_once UserModel.php and corrected \\UserModel() namespace error.
✅ Fix 2 — DoctorController Missing Import
Same issue in DoctorController::handleProfile(). Missing require_once UserModel.php and \\UserModel() backslash namespace bug corrected.
✅ Fix 3 — Admin Notification Spam
Admin dashboard was creating duplicate low-stock notifications on every page load. Removed the loop — alerts are already shown as HTML banners.
✅ Fix 4 — StudentModel Missing JOIN Columns
findByUserId() was missing u.gender and u.date_of_birth from the JOIN. Student profile page would silently show empty fields without this fix.

Setup Guide
Install in 4 Steps
1

Extract ZIP into XAMPP htdocs

Extract the ZIP so the folder is at C:\xampp\htdocs\medibase_uni\ (Windows) or /Applications/XAMPP/htdocs/medibase_uni/ (Mac).

2

Import the SQL schema

Open phpMyAdmin → click Import → select medibase_enhanced.sql → click Go. This creates the medibase_uni database with all tables and seed data.

3

Configure database (if needed)

Edit config/db.php if your MySQL password is not blank. The default config uses root with no password — standard XAMPP setup.

4

Open in browser

Start Apache & MySQL in XAMPP, then visit http://localhost/medibase_uni/public/. Log in with any seeded account. Default password for all: Admin@1234
